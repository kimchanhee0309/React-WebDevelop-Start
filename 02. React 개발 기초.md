# 1. JSX

<details>
  <summary>(1) JSX란?</summary>

  * JSX = **JavaScript 속에서 HTML처럼 쓸수 있는 문법**
</details>

<details>
  <summary>(2) JSX의 3가지 기본 규칙</summary>

  * **규칙 1: 반드시 하나의 태그로 감싸기**
    ```JavaScript
    // 에러! 태그 두 개가 나란히 있음
    return (
      <h1>제목<h1>
      <p>내용</p>
    );

    // OK - div로 감쌌음
    return (
      <div>
        <h1>제목</h1>
        <p>내용</p>
      </div>
    );

    // 감싸는 div가 싫다면? 프래그먼트를 통해 해결 가능
    ```
    
  * **규칙 2: HTML 속성은 camelCase로**
    * JSX와 HTML은 비슷하지만, **속성 이름이 약간 다름**
      
  |HTML|JSX|
  |---|---|
  |`class`|`className`|
  |`for`|`htmlFor`|
  |`onclick`|`onClick`|
  |`onchange`|`onChange`|
  |`tabindex`|`tabIndex`|

  ```JavaScript
  // HTML 스타일
  <div class="container" onclick="handleClick()">

  // JSX 스타일
  <div className="container" onClick={handleClick}>

  // class는 JS에서 이미 예약어이기 때문에, 겹치지 않게 바꾸는 것
  ```

  * **규칙 3: 모든 태그는 반드시 닫기**
    ```JavaScript
    // X, HTML에선 가능했지만 JSX에선 에러
    <br>
    <img src="photo.jpg">
    <input type="text">

    // O, 셀프 클로징(/>)으로 닫기
    <br />
    <img src="photo.jpg" />
    <input type="text" />
    ```
</details>

<details>
  <summary>(3) JSX에서 주석 쓰기</summary>

  * JSX 안에선 주석을 `{/* */}` 형태로 씀
    ```JavaScript
    function App() {
      return (
        <div>
        {/* 이렇게 주석을 쓴다 */}
        <h1>제목</h1>

        {/*
          여러 줄 주석도
          이렇게 가능해요
        */}
        <p>내용</p>
      </div>
      };
    }
    ```
</details>

# 2. 프래그먼트(Fragment)

<details>
  <summary>(1) div로 감싸면 생기는 문제</summary>

  * **문제 1: 불필요한 DOM이 쌓임**
    * `<div>` 감싸기를 반복하면 실제 HTML 구조는 다음과 같이 됨
      ```JavaScript
      <div>
        <div>
          <div>
            <div>
              <h1>안녕</h1>
            </div>
          </div>
        </div>
      </div>
      ```
      * **의미 없는 `<div>` 껍데기가 잔뜩 쌓임**
      
  * **문제 2: 특정 HTML 구조에선 아예 에러**
    * `<table>` 같이 **규칙이 엄격한 구조**에선 `<div>`를 끼워넣으면 HTML이 깨짐
      ```JavaScript
      // 잘못된 구조
      function TableRow() {
        return (
          <div>    {/* 여기 div 때문에 테이블이 망가짐! */}
            <td>이름</td>
            <td>나이</td>
          </div>
        };
      }
      ```
      ```JavaScript
      <!-- 결과: HTML 표준 위반! -->
      <table>
	      <thead>
	        <tr>
	          <div>   ← table 안에 div가 들어갈 수 없어요!
	            <td>이름</td>
	            <td>나이</td>
	          </div>
	        </tr>
	      </thead>
      </table>
      ```
</details>


<details>
  <summary>(2) Fragment = 투명한 포장지</summary>

  * Fragment는 **투명한 포장지**임
  * 여러 물건을 **묶어주긴 하지만**, 실제 DOM에선 **보이지 않음**
  * `<div>`가 택배 박스라면, Fragment는 **랩으로 살짝 감싸는 느낌**
</details>


<details>
  <summary>(3) Fragment 쓰는 2가지 방법</summary>

  * **방법 1: 빈 태그 `<> </>`(가장 많이 씀)⭐**
    ```JavaScript
    function TableRow() {
      return (
        <>
          <td>이름</td>
          <td>나이</td>
        </>
      };
    }
    ```
  * **방법 2: `<React.Fragment>`(풀네임)**
    ```JavaScript
    import React from 'react';

    function TableRow() {
      return (
        <React.Fragment>
          <td>이름</td>
          <td>나이</td>
        </React.Fragment>
      };
    }
    ```

  * **✨ Fragment를 쓴 결과**
    ```JavaScript
    <!-- 이제 HTML 구조가 깨끗해요! -->
    <table>
      <tr>
        <td>이름</td>
        <td>나이</td>
      </tr>
    </table>
    ```
    * `<div>`도, Fragment도 **결과 HTML엔 흔적이 없음**
</details>


<details>
  <summary>(4) Fragment vs div - 언제 뭘 쓸까?</summary>

  |상황|추천|
  |---|---|
  |스타일(CSS)이나 클래스가 필요할 때|`div`|
  |id, onClick 등 속성이 필요할 때|`div`|
  |단순히 그룹화만 필요할 때|Fragment`<>`|
  |테이블, 리스트 등 HTML 구조가 엄격할 때|Fragment`<>`|

  * **간단한 판단 기준**
    * **이 감싸는 태그에 뭔가 필요한가?**
      * 스타일/속성이 필요해 -> `<div>`
      * 그냥 묶기만 하면 돼 -> `<> </>`

  * **⚠️ 예외: Key가 필요할 땐 `React.Fragment`**
    * **딱 한 가지 경우**에만 빈 태그 `<>` 대신 **풀네임 `<React.Fragment>`**를 써야 함
    * 바로 **리스트에 key가 필요할 때**임
      ```JavaScript
      // 빈 태그는 key를 받을 수 없음
      users.map(user => (
        <key={user.id}>  {/* 이렇게 못 씀! */}
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </>
      ))
      
      // key가 필요하면 풀네임 Fragment 사용
      users.map(user => (
        <React.Fragment key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </React.Fragment>
      ))
      ```

  * **정리**
    
  |문법|용도|
  |---|---|
  |`<> </>`⭐|거의 모든 경우(권장)|
  |`<React.Fragment>`|key가 필요한 예외 상황|
  |`<div>`|스타일/속성이 필요할 때|
      
</details>

# 3. JSX에서 JavaScript 사용하기

<details>
  <summary>(1) 변수와 상수 - 기본 중의 기본</summary>

  ```JavaScript
  function UserCard() {
    const userName = '김철수';
    const userAge = 28;

    return (
      <div>
        <h2>{userName}</h2>
        <p>나이: {userAge}세</p>
      </div>
    };
  }

  // 가장 흔한 사용법
  // JSX 바깥에서 변수를 선언하고 안에서 꽂아 넣기
  ```
</details>

<details>
  <summary>(2) 객체 사용 - 점(.)으로 접근</summary>

  ```JavaScript
  function UserProfile() {
    const user = {
      name: '이영희',
      age: 32,
      job: '디자이너'
    };

    return (
      <div>
        <h2>{user.name}</h2>
        <p>{user.age}세, {user.job}</p>
      </div>
    );
  }
  ```
  * 객체의 속성은 **점 표기법(`user.name`)** 으로 접근
</details>

<details>
  <summary>(3) 함수 호출 - 복잡한 로직은 함수로</summary>

  ```JavaScript
  function Greeting() {
    function getGreeting(hour) {
      if (hour < 12) return '좋은 아침입니다';
      if (hour < 18) return '좋은 오후입니다';
      return '좋은 저녁입니다';
    }

    const currentHour = new Date().getHours();

    return <h1>{getGreeting(currentHour)}</h1>;
  }
  ```
  * **조건이 복잡하면 함수로 빼야함**
  * JSXsms **뭘 보여줄지**에 집중하고, **어떻게 계산할지**는 함수에 맡기는 게 깔끔함
</details>

<details>
  <summary>(4) 조건 표현식 - 삼항 연산자</summary>

  ```JavaScript
  function StatusBadge({ status }) {
    return (
      <span className={status === 'active' ? 'badge-green' : 'badge-gray'}>{status === 'active' ? '활성' : '비활성'}</span>
    );
  }
  ```
</details>

<details>
  <summary>(5) 배열 메서드 - map과 filter</summary>

  * 리스트를 화면에 뿌릴 때 가장 많이 쓰는 패턴
  * **기본 - `map` - 배열의 각 요소를 JSX로 변환**
    ```JavaScript
    function FruitList() {
      const fruits = [
        { id: 1, name: '사과' },
        { id: 2, name: '바나나' },
        { id: 3, name: '오렌지' }
      ];

      return (
        <ul>
          {fruits.map(fruit => (
            <li key={fruit.id}>{fruit.name}</li>
          ))}
        </ul>
      );
    }
    ```

  * **응용 - `filter` + `map`체이닝**
    ```JavaScript
    function UserList({ users }) {
      return (
        <ul>
          {users
            .filter(user => user.active)  // 활성 사용자만
            .map(user => (                // JSX로 변환
              <li key={user.id}>{user.name}</li>
            ))}
        </ul>
      );
    }
    ```
  * **⚠️ 안티패턴 #1: `key{index}`**
    ```JavaScript
    // X index를 key로 쓰면 안됨
    {fruits.map((fruit, index) => (
      <li key={index}>{fruit}</li>
    ))}
    ```
  * **개선: 고유한 ID를 key로**
    ```JavaScript
    // 각 항목의 고유 id를 key로
    {fruits.map(fruit => (
    	<li key={fruit.id}>{fruit.name}</li>
    ))}
    ```
</details>

<details>
  <summary>(6) 템플릿 리터럴 - 문자열 조합</summary>

	```JavaScript
	function ProductCard({ product }) {
		return (
			<div className={`product-card ${product.featured ? 'featured' : ''}`}>
				<h3>{product.name}</h3>
			</div>
		);
	}
	```
	
  * **⚠️ 안티패턴: 불필요한 템플릿 리터럴**
    ```JavaScript
    <p>{`₩${product.price.toLocaleString()}`}</p>
    <p>{`재고: ${product.stock}개`}</p>
    ```

  * **개선: JSX의 자연스러운 흐름 활용**
    ```JavaScript
    <p>₩{product.price.toLocaleString()}</p>
    <p>재고: {product.stock}개</p>
    ```
</details>

<details>
  <summary>(7) 꼭 피해야 할 패턴: 즉시 실행 함수(IIFE)</summary>

  * **안티패턴: JSX 안에서 IIFE**
    ```JavaScript
    function Dashboard({ score }) {
    	return (
    		<div>
    			<p>점수: {score}</p>
    			<p>등급: {
    				(() => {
    					if (score >= 90) return 'A';
    					if (score >= 80) return 'B';
    					if (score >= 70) return 'C';
    					return 'D';
    				})()
				}</p>
    		</div>
    	);
    }
    ```
    * **문제점**
	  * **읽기 지옥**: `(() => {})()` 괄호 지옥
      * **디버김 어려움**: 함수에 이름이 없어서 에러 추적이 힘듦
      * **재사용 불가**: 같은 로직을 다른 곳에서 쓸 수 없음

  * **개선: 함수로 분리**
    ```JavaScript
    function getGrade(score) {
    	if (score >= 90) return 'A';
    	if (score >= 80) return 'B';
    	if (score >= 70) return 'C';
    	return 'D';
    }

    function Dashboard({ score }) {
    	return (
    		<div>
    			<p>점수: {score}</P>
    			<P>등급: {getGrade(score)}</p>
    		</div>
    	);
    }
    ```
</details>

# 4. 컴포넌트(Component)

<details>
  <summary>(1) 컴포넌트란?</summary>

* **한 줄 정의: 화면을 구성하는 독립적인 부품**
* **쇼핑몰 하나를 컴포넌트로 쪼개보면?**
  ```
  🛍️ 쇼핑몰 페이지
  ├── 📋 Header (로고 + 검색창 + 장바구니)
  ├── 📦 ProductList
  │   ├── 🃏 ProductCard (상품 1)
  │   ├── 🃏 ProductCard (상품 2)
  │   └── 🃏 ProductCard (상품 3)
  └── 📜 Footer
  ```
  * 각 부품(컴포넌트)는 **독립적**이고, **재사용 가능함**
  * `ProductCard` 하나를 잘 만들어두면, 상품이 100개든 1000개든 끼워넣으면 됨

  * **컴포넌트의 3대 장점**

	|장점|설명|
	|---|---|
	|**🔄 재사용성**|한 번 만들면 여러 곳에서 쓸 수 있음|
	|**🛠️ 유지보수**|버그 나면 해당 컴포넌트만 고치면 됨|
	|**👥 협업**|팀원들이 각자 다른 컴포넌트 담당 가능|

</details>

<details>
  <summary>(2) 컴포넌트 만들기</summary>

* **컴포넌트 = JSX를 반환하는 함수**
  ```JavaScript
  function Welcome() {
  	return <h1>환영합니다!</h1>;
  }
  ```
  
* **쓸 때는 HTML 태그처럼!**
  ```JSX
  function App() {
  	return (
  		<div>
  			<Welcome />		{/* ← 내가 만든 컴포넌트를 태그처럼! */}
  			<Welcome />	  {/* 재사용도 가능! */}
  			<Welcome />
  	);
  }
  ```
	
</details>

<details>
  <summary>(3) 절대 규칙: PascalCase로 시작하기</summary>

* 컴포넌트 이름은 **반드시 대문자**로 시작해야 함(타협 불가)
  ```JavaScript
  // O
  function Welcome() { }
  function UserProfile() { }
  function ProductCard() { }

  // X
  function welcome() { }				// React가 HTML 태그로 착각
  function userProfile() { }	  // 렌더링 안 됨
  ```
</details>

<details>
  <summary>(4) 컴포넌트 분리하기 - Before/After</summary>

* **Before: 한 파일에 전부(지옥)**
  ```JSX
  function App() {
  	return (

  				<h1>My Blog</h1>

  			<h1>My Blog</h1>
  			<nav>

  				<a href="/about">About</a>
  			</nav>

  function Article() {
  	return (

</details>

# 5. Props⭐️

* **학습목표**: 컴포넌트에 **데이터를 전달**해서, 같은 컴포넌트를 **다양하게 활용**
<details>
  <summary>(1) Props란?</summary>

* **Props** = **Prop**erties의 줄임말
* **한 줄 정의**: **부모 컴포넌트**가 **자식 컴포넌트**에게 전달하는 **데이터**
* **기본 예제**
  ```JSX
  // 자식 컴포넌트
  function Greeting(props) {
  	return <h1>안녕하세요, {props.name}님!</h1>;
  }

  // 부모 컴포넌트
  function App() {
  	return (
  		<div>
  			<Greeting name="홍길동" />
  			<Greeting name="김철수" />
  			<Greeting name="이영희" />
  		</div>
  	);
  }
  ```
  * **결과**
    ```
    안녕하세요, 홍길동님!
    안녕하세요, 김철수님!
    안녕하세요, 이영희님!
    ```
</details>

<details>
  <summary>(2) 구조 분해 할당 - 더 깔끔하게</summary>

* **기본 방식**
  ```JSX
  function UserCard(props) {
  	return (
  		<div>
  			<h2>{props.name}</h2>
  			<p>{props.age}세</p>
  			<p>{props.job}</p>
  	);
  }
  ```

* **구조 분해 할당(실무에선 이게 표준임)**
  ```JSX
  function UserCard({ name, age, job }) {
  	return (
  		<div>
  			<h2>{name}</h2>
  			<p>{age}세</p>
  			<p>{job}</p>
  		</div>
  	);
  }

  // 중괄호{}로 받으면 각 속성을 변수처럼 바로 쓸 수 있음
  ``` 
</details>

<details>
  <summary>(3) 다양한 타입의 Props 전달하기</summary>

* **📝 문자열 - 따옴표만**
  ```JSX
  <Button text="클릭하세요" />
  ```

* **🔢 숫자, 불린, 배열, 객체 - 중괄호 {}**
  ```JSX
  <Counter count={0} />
  	{/* 숫자 */}
  <Modal isOpen={true} />
  	{/* 불린 */}
  <TagList tags={['React', 'JS', 'CSS']} />
  	{/* 배열 */}
  <Profile user={{ name: '홍길동', age: 25 }} />
  	{/* 객체 */}

  // 문자열만 따옴표, 나머진 전부 중괄호{}
  ```

* **함수도 전달 가능**
  ```JSX
  <Button onClick={() => alert('클릭!')} />
  <Form onSubmit={handleSubmit} />
  ```
</details>

<details>
  <summary>(4) Props 기본값 설정하기</summary>

* Props를 **안 넘겼을 때 쓸 기본값**을 정해둘 수 있음
* **기본 매개변수(모던 방식, 권장)⭐**
  ```JSX
  function Button({ text = '버튼', color = 'blue' }) {
  	return <button style={{ color }}>{text}</Button>;
  }

  // 사용
  <Button />												{/* "버튼", 파란색 */}
  <Button text="클릭" />							{/* "클릭", 파란색 */}
  <Button text="클릭" color="red" /> {/* "클릭", 빨간색 */}
  ```
</details>

<details>
  <summary>(5) Spread 연산자로 한 번에 전달하기</summary>

* 객체를 **통째로** 넘기고 싶을 때 사용
	```JSX
 	function App() {
 		const user = {
 			name: '홍길동',
 			age: 25,
 			job: '개발자'
 	};

 	// 일일이 전달
 	return <UserCard name={user.name} age={user.age} job={user.job} />;

 	// Spread 연산자
 	return <UserCard {...user} />;
 	```

* **안티패턴: 무분별한 Spread 남용**
  ```JSX
  // 부모가 받은 props를 묻지도 따지지도 않고 자식에게 몽땅 전달
  function ParentCard(props) {
  	return <ChildCard {...props} />;
  }
  ```
  * **문제점**
  	* **자식이 어떤 props를 받는지** 코드만 봐선 모름
   	* **불필요한 props가 섞여** 들어가서 디버깅 지옥
    * 자식 컴포넌트가 바뀌어도 **부모 코드는 조용히 깨짐**

	* **개선: 필요한 것만 명시적으로**
    ```JSX
    // 어떤 props를 넘기는지 명확
    function ParentCard({ name, age }) {
    	return <ChildCard name={name} age={age} />;
		}
    ```
</details>

<details>
  <summary>(6) Props는 읽기 전용</summary>

* **가장 중요한 규칙**: 자식은 **Props를 절대 수정할 수 없음**
  ```JSX
  // 절대 금지
  function Button({ text }) {
  	text = '새 텍스트';	// 수정 시도! React의 규정 위반
  	return <button>{text}</button>;
  }
  ```

* **값을 가공하고 싶다면? 새 변수에**
  ```JSX
  function Button({ text }) {
  	const upperText = text.toUpperCase();	// 새 변수로
  	return <button>{upperText}</button>;
  }
  ```
</details>

<details>
  <summary>(7) 실무 네이밍 관례</summary>

* **Props 이름은 camelCase**
  ```JSX
  // O
  <UserCard userName="홍길동" isActive={true} />

  // X
  <UserCard user_name="홍길동" IsActive={true} />
  ```
  
* **이베트 핸들러는 on + 동사**
  ```JSX
  // O
  <Button onClick={handleClick} />
  <Form onSubmit={handleSubmit} />
  <Input onChange={handleChange} />

  // X
  <Button clickHandler={handleClick} />
  <Button whenClicked={handleClick} />
  ```
</details>

<details>
  <summary>(8) 정리</summary>

|개념|요약|
|---|---|
|**Props란?**|부모 -> 자식으로 전달하는 데이터|
|**기본 사용**|`<Component propName="값" />`|
|**구조 분해**|`function({ name, age })` ⭐ 권장|
|**중괄호 규칙**|문자열은 `""`, 나머진 `{}`|
|**기본값**|`{ name = '기본값' }`⭐(defaultProps X)|
|**Spread**|`<C {...user} />` (신중하게)|
|**읽기 전용**|Props는 절대 수정 금지|


* **기억할 3가지 핵심**
	* 1. **Props는 한 방향** (부모 -> 자식)
  * 2. **Props는 수정 불가** (읽기 전용)
  * 3. **defaultProps는 이제 안씀** (기본 매개변수만)
</details>

# 6. children⭐️

* **학습 목표**: 컴포넌트를 **박스**처럼 만들고, **그 속에 내용을 자유롭게 담는** 법

<details>
  <summary>(1) children이란?</summary>

* **한 줄 정의: 컴포넌트 태그 사이에 들어가는 내용**
  ```JSX
  <Button>클릭하세요</Button>		{/* "클릭하세요" = children */}
  <Card>
  	<h2>제목</h2>							{/* 이 JSX 전체 = children */}
  	<p>내용</p>
  </Card>
  ```
</details>

<details>
  <summary>(2) children 받아쓰기</summary>

* `children`은 **특별한 이름의 Props**임
* 다른 Props랑 똑같이 구조 분해로 받으면 끝
  ```JSX
  function Button({ children }) {
  	return <button>{children}</button>;
  }

  // 사용
  <Button>저장하기</Button>
  <Button>삭제하기</Button>
  <Button>⭐ 특별 버튼</Button>
  ```
  * **`children`이라는 이름은 고정임**
	
</details>

<details>
  <summary>(3) Props vs children - 언제 뭘 쓸까?</summary>

* **Props: 정해진 데이터 전달**
  ```JSX
  <ProductCard
  	name="사과"
  	price={1000}
  	imageUrl="apple.jpg"
  />
  ```
  * 각 props가 **역할이 정해져 있음** (name, price, imageUrl)
  * 주로 **값(문자열, 숫자, 불린)** 전달
  * 컴포넌트마다 **미리 정의된 슬롯**에 꽂아넣는 느낌

* **children: 자유로운 내용물 전달**
  ```JSX
  <Card>
  	<h2>무엇이든</h2>
  	<p>넣어도 되는</p>
  	<img src"..." />
  	<Button>자유 공간</Button>
  </Card>
  ```
  * **정해진 구조가 없음**
  * 텍스트, JSX, 다른 컴포넌트 **뭐든 OK**
  * 컴포넌트를 **범용 컨테이너**로 만들 때 유용

* **판단 기준**
|상황|추천|
|---|---|
|값만 넘기면 됨(이름, 가격)|**Props**|
|필수 데이터(사용자 정보 등)|**Props**|
|여러 요소/복잡한 JSX|**children**|
|버튼 텍스트, 제목 같은 단순 텍스트|**children 권장**|
</details>

# 7. State⭐️
* **학습 목표**: 시간에 따라 **변하는 값**을 React답게 다루는 법을 익힘

<details>
  <summary>(1) State란?</summary>

  * **한 줄 정의**: 컴포넌트가 **기억하는 값**. 변하면 **화면도 자동 업데이트**

  ```JSX
  const [count, setCount] = userState(0);
  ```
  * `count` = 현재 **채널 번호**(지금 값)
  * `setCount` = **채널 바꾸는 버튼**(값 바꾸는 도구)
  * `useState(0)` = 리모컨 초기 설정 (**처음 값은 0**)
  * **중요** : 반드시 `setCount`로 바꿔야 React가 알아차림

  * **Props vs State**
    
  | |Props|State|
  |---|---|---|
  |**누가 관리?**|부모 컴포넌트가 전달|**자기 자신이 관리**|
  |**수정 가능?**|X(읽기 전용)|O(가능, setter로만)|
  |**변하면?**|부모가 다시 내려줘야|**자동으로 화면 업데이트**|
  |**비유**|받은 택배|내 비밀 노트|
  
</details>

<details>
  <summary>(2) useState 사용하기</summary>

  * **기본 문법**
    ```JSX
    import { userState } from 'react';    // <- 꼭 import!

    function Counter() {
      const [count, setCount] = useState(0);

      return (
        <div>
          <p>카운트: {count}</p>
          <button onClick={() => setCount(count + 1)}>
            증가
          </button>
        </div>
      );
    }
    ```
    ```JSX
    const [count, setCount] = useState(0);
    ```
  |부분|의미|
  |---|---|
  |`useState(0)`|"0부터 시작하는 state 만들어줘"|
  |`[count, setCount]`|"값은 `count`로, 변경 함수는 `setCount`로 받을게"|
  |`const`|재할당 방지(setCount로만 바꾸도록)|
    
</details>

<details>
  <summary>(3) 여러 State 함께 쓰기</summary>

  ```JSX
  function LoginForm() {
    const [email, setEmail] = useState('');
    const [passwor, sestPassword] = useState('');
    const [isLoggedIn, setIsLoggedIn] = useState(false);

    function handleSubmit(e) {
      e.preventDefault();
      // 로그인 로직..
      setIsLoggedIn(true);
    }

    return (
      <form onSubmit={handleSubmit}>
        <input
          type="email"
          value={email}
          onChange={(e) => setEmail(e.target.value)}
        />
        <input
          type="password"
          value={password}
          onChange={(e) => setPassword(e.target.value)}
        />
        <button>로그인</button>
      </form>
    );
  }
  ```
  ```JSX
  <input
    value={email}
    onChange={(e) => setEmail(e.target.value)}
  />
  ```
  * `value={email}`: **입력창이 보여줄 값은 `email` state**
  * `onChange`: **입력이 바뀔 때마다 setEmail 호출**
  * `e.target.value`: **사용자가 방금 입력한 값**
</details>

<details>
  <summary>(4) 중요한 패턴: 함수형 업데이트</summary>

  * **방법 1: 직접 값 전달(기본)**
    ```JSX
    setCount(count + 1);
    ```

  * **방법 2: 함수 전달(실무에선 이게 더 안전)⭐**
    ```JSX
    setCount(prev => prev + 1);  // prev = 이전 값
    ```

  * **두 방법의 차이**
    ```JSX
    function handleClick() {
      // 방법 1: 직접 값
      setCount(count + 1);
      setCount(count + 1);
      setCount(count + 1);
      // -> 1만 증가

      // 방법 2: 함수 전달
      setCount(prev => prev + 1);
      setCount(prev => prev + 1);
      setCount(prev => prev + 1);
      // -> 3 증가
    }
    ```

  * **값이 다르게 나온 이유?**
    * **방법 1의 함정**
      ```JSX
      // count가 0일 때 버튼 클릭
      setCount(count + 1);  // setCount(0 + 1) = 1
      setCount(count + 1);  // setCount(0 + 1) = 1    <- 같은 count!
      setCount(count + 1);  // setCount(0 + 1) = 1    <- 역시 같은 count!
      // 결국 1로 덮어쓰기만 됨
      ```
      * `count`는 리렌더링 전까진 **0으로 고정**임
      * 3번 호출해도 전부 **같은 계산(`0 + 1`)** 해버림

    * **방법 2의 똑똑함**
      ```JSX
      setCount(prev => prev + 1);  // prev=0 -> 1
      setCount(prev => prev + 1);  // prev=1 -> 2
      setCount(prev => prev + 1);  // prev=2 -> 3
      ```
      * `prev`는 **이전 업데이트 결과**를 넘겨받음

    * **언제 어떤 걸 써야 하나?**
    
  |상황|추천|
  |---|---|
  |한 이벤트에서 **딱 한 번** 호출|직접 값(`setCount(count + 1)`) OK|
  |**여러 번 호출**하거나 비동기 작업 후|**함수 전달** 필수|
  |**이전 값에 기반한 업데이트**|**함수 전달** 권장|
       
      
</details>

<details>
  <summary>(5) 토글 패턴</summary>

  ```JSX
  function ToggleButton() {
    const [isOn, setIsOn] = useState(false);

    return (
      <button onClick={() => setIsOn(prev => !prev)}>
        {isOn ? 'ON' : 'OFF'}
      </button>
    );
  }
  ```
  * `prev => !prev` : "이전 값의 반대"로 바꾸기. 함수형 업데이트 활용의 정석
</details>

# 8. 참조형 State⭐️
* **학습 목표**: **객체와 배열**을 State로 쓸 때의 **함정**을 피하고, **불변성**을 지키며 안전하게 업데이트 함
  
<details>
  <summary>(1) 불변성(Immutability) - 오늘의 절대 원칙</summary>

  * **불변성 = 기존 값을 수정하지 않고, 항상 새로 만들기**
    ```JSX
    // X, 기존 객체 수정(React가 못 알아챔)
    user.name = '김철수';
    setUser(user);

    // O, 새 객체 생성(React가 알아챔)
    setUser({ ...user, name: '김철수' });
    ```
</details>

<details>
  <summary>(2) 객체 State 다루기</summary>

  * **기본 패턴: `{ ...기존객체, 바꿀속성: 새값 }`**
    ```JSX
    function UserProfile() {
      const [user, setUser] = userState({
        name: '홍길동',
        age: 25,
        email: 'hong@example.com'
      });

      function updateName(newName) {
        setUser({
          ...user,        // 기존 속성 모두 복사
          name: newName   // name만 덮어쓰기
        });
      }

      // ...
    }
    ```
  * **`...user`의 역할**
    ```JSX
    setUser({ ...user, name: '김철수' });
    // ...user : "기존 속성을 전부 펼쳐서 복사해줘"
    ```
</details>

<details>
  <summary>(3) 배열 State 다루기 - 3대 패턴</summary>

  * **공통 준비**
    ```JSX
    const [todos, setTodos] = userState([
      { id: 1, text: '운동하기', done: false },
      { id: 2, text: '책 읽기', done: false }
    ]);
    ```

  * **+ 패턴 1: 추가 - Spread `[...arr, 새항목]`**
    ```JSX
    function addTodo(newTodo) {
      setTodos([...todos, newTodo]);  // 새 배열
    }
    ```
    * **절대 하지 말 것**
    ```JSX
    todos.push(newTodo);  // 원본 수정, React가 모름
    setTodos(todos);
    ```
    
  * **- 패턴 2: 삭제 - `filter(조건)`**
    ```JSX
    function deleteTodo(id) {
      setTodos(todos.filter(todo => todo.id !== id));
      // filter : "id가 일치하지 않는 것만 남겨줘"
    }
    ```
    * `filter`는 **조건에 맞는 항목들로 새 배열**을 만듦. 원본을 수정하지 않아서 완벽함
    
  * **패턴 3: 수정 - `map()` + 조건**
    ```JSX
    function toggleTodo(id) {
      setTodos(todos.map(todo =>
        todo.id === id
          ? { ...todo, done: !todo.done }  // 해당 항목만 바꾸기
          : todo                           // 나머진 그대로
      ));
    }

    // 해석: 각 todo를 살펴보면서, id가 일치하면 done을 뒤집고, 아니면 그대로 두세요.
    ```
    
  * **3대 패턴 암기표**

|작업|패턴|핵심 함수|
|---|---|---|
|추가|`[...arr, 새항목]`|spread|
|삭제|`arr.filter(조건)`|filter|
|수정|`arr.map(...)`+조건 분기|map|

  * **공통점**: 셋 다 **새 배열을 만들어 교체함**. 원본은 절대 안건드림

  * **완성 예제: Todo 리스트**
    ```JSX
    function TodoList() {
      const [todos, setTodos] = useState([
        { id: 1, text: '운동하기', done: false },
        { id: 2, text: '책 읽기', done: false }
      ]);

    function toggleTodl(id) {
      setTodos(todos.map(todo =>
        todo.id === id ? { ...todo, done: !todo.done } : todo
      ));

    function deleteTodo(id) {
      setTodos(todos.filter(todo => todo.id !== id));
    }

    return (
      <div>
        {todos.map(todo => (
          <div
            key={todo.id}  // index 말고 반드시 고유 id!
            style={{ textDecoration: todo.done ? 'line-through' : 'none' }}
          >
            <input>
              type="checkbox"
              checked={todo.done}
              onChange={() => toggleTodo(todo.id)}
            />
            {todo.text}
            <button onClick={() => deleteTodo(todo.id)}>삭제</button>
          </div>

    ```

# 10. React가 렌더링하는 방식

<details>
  <summary>(1) 리액트에서 말하는 "렌더링"이란?</summary>

  * **렌더링=컴포넌트 함수 실행**
    ```JSX
    function Counter() {
      const [count, setCount] = useState(0);
      return <button>{count}</button>;
    }
    ```
    * **렌더링된다** = React가 **`Counter()`를 호출**한다는 뜻
    * 렌더링은 계산이고, 화면 업데이트는 반영이기 때문에, 렌더링은 화면이 실제로 바뀌는 것이 아님!
</details>

<details>
  <summary>(2) 두 단계로 나뉨</summary>

  * **1. Render Phase(계산 단계) - "설계도 그리기"**
    * 컴포넌트 함수 호출
    * JSX 반환
    * 가상 DOM 생성
    * 이전 가상 DOM과 비교
   
  * **2. Commit Phase(적용 단계) - "실제 건설"**
    * 실제 DOM에 변경사항 반영
    * 화면에 보여지는 순간
</details>

<details>
  <summary>(3) 언제 렌더링 될까?</summary>

  ```JSX
  // 1. 처음 화면에 나타날 때(최초 렌더링)
  <App />

  // 2. State가 바뀔 때
  setCount(count + 1);  // -> 리렌더링

  // 3. Props가 바뀔 때
  <UserCard name={newName} />  // name이 바뀌면 -> 리렌더링

  // 4. 부모가 렌더링될 때
  function Parent() {
    return (
      <div>
        <Child />    {/* 부모가 렌더링되면 자식도 같이! */}
      </div>
    );
  }
  ```
  * 부모가 렌더링되면 **자식은 Props가 안 바뀌어도 리렌더링됨**
</details>

<details>
  <summary>(4) 배치 업데이트(Batching) - 영리한 모아 처리</summary>

  * 여러 setter를 **한 번에 처리**하는 React의 최적화 기능
    ```JSX
    function Counter() {
      const [count, setCount] = useState(0);
      const [name, setName] = useState('');

      function handleClick() {
        setCount(count + 1);      // 렌더링 예약
        setName('업데이트됨');       // 렌더링 예약
        setCount(count + 2);      // 렌더링 예약
        // -> 이벤트가 끝난 뒤 "한 번만" 렌더링!
      }

      console.log('렌더링!');    // 한 번만 출력됨
    }
    ```
</details>

<details>
  <summary>(5) 주의해야 할 패턴</summary>

  * **❌ 안티패턴 1: 렌더링 중 State 변경**
    ```JSX
    // X, 무한 루프!
    function Counter() {
      const [count, setCount] = useState(0);

      setCount(count + 1);  // 렌더링 중 setter 호출 -> 또 렌더링 ....

      return <p>{count}</p>;
    }
    ```JSX
  * **❌ 안티패턴 2: 렌더링 안에서 사이드 이펙트**
    ```JSX
    // X, 렌더링마다 API 호출
    function UserProfile() {
      const [user, setUser] = useState(null);

      fetch('/api/user').then(res => res.json()).then(setUser);
      // 렌더링마다 호출 -> 무한 호출 폭탄

      return <div>{user?.name}</div>
    ```
    * **해결**: API 호출 같은 **사이트 이펙트**는 `useEffect`에서!
    
  * **❌ 안티패턴 3: 컴포넌트를 다른 컴포넌트 안에 정의**
    ```JSX
    // X, 매 렌더링마다 Child를 새로 만듦
    function Parent() {
      function Child() {    // 내부에 정의
        return <div>자식</div>;
      }

      return <Child />;
    }

    // O, 밖에 정의
    function Child() {
      return <div>자식</div>;
    }

    function Parent() {
      return <Child />;
    }
    ```
    * **문제점**: Parent가 렌더링될 때마다 **Child가 매번 새로운 컴포넌트**로 취급됨. State가 초기화되고, 성능도 나빠짐
</details>

# 11. 인라인 스타일
* **학습 목표**: React에서 **스타일을 직접 적용**하는 법을 배우고, **언제 쓰고 언제 안 써야 하는지** 명확한 기준을 세운다.

<details>
  <summary>(1) 인라인 스타일 기본 문법</summary>

  * HTML에선
    ```HTML
    <button style="background-color: blue; color: white;">클릭</button>
    ```
    
  * JSX에선
    ```JSX
    <button style={{
      backgroundColor: 'blue',
      color: 'white'
    }}>
      클릭
    </button>
    ```

  * **JSX 인라인 스타일 3대 규칙
|규칙|HTML|JSX|
|---|---|---|
|속성명|`background-color(kebab-case)|`backgroundColor(camelCase)|
|값|그냥 쓰기|**문자열로**(`blue`)|
|단위|`10px`|`10` 또는 `'10px'`(숫자면 자동 px)|

 * **예제**
   ```JSX
   <div style={{
     backgroundColor: '#f0f0f0',      // background-Color -> backgroundColor
     fontSize: 16,                    // 16 -> 자동으로 16px
     marginTop: '20px',               // 문자열로 단위 명시도 OK
     padding: '10px 20px',            // 복합값은 문자열 필수
     borderRadius: 8
   }}>
     내용
   </div>
   ```
</details>

<details>
  <summary>(2) 스타일 객체를 분리하기</summary>

  * **안 좋은 예시**
    ```JSX
    function Button() {
      return (
        <button style={{
          backgroundColor: 'blue',

          border:  'none',

<details>
  <summary>(4) 인라인 스타일의 한계</summary>

  * **🚫1. `:hover`, `:focus` 같은 가상 선택자 불가**
    ```JSX
    <button style={{
      backgroundColor: 'blue',
      ':hover': { backgroundColor: 'darkblue' } // 안 먹힘
    }}>
    ```
    
  * **🚫2. 미디어 쿼리 불가(반응형 디자인)**
    ```JSX
    <div style={{
      '@media (max-width: 768px)': { fontSize: 14 }
    }}>
    ```
    
  * **🚫3. 애니메이션 키프레임 불가**
    * `@keyframes`를 정의할 수 없음
      
  * **🚫4. 재사용하기 번거로움**
    * 같은 스타일을 **여러 컴포넌트에서 공유**하려면 파일로 분리하고 import해야 함

  * **🚫5. 성능에 미세하게 안 좋음**

  <summary>(1) className 기본</summary>

<details>
  <summary>(3) 조건부 클래스 3가지 패턴</summary>

  * **패턴 1: 삼항 연산자(단일 조건)**
    ```JSX
    function TodoItem({ todo }) {
      return (
        <div className={`todo-item ${todo.done ? 'done' : ''}`}>
          {todo.text}
        </div>
      );
    }

    // todo.done이 true면 -> todo-item done
    // false면 -> todo-item
    ```
    
  * **패턴 2: && 연산자(조건부 추가)**
    ```JSX
    function Button({ disabled }) {
      return (
        <button className={`btn ${disabled && 'btn-disabled'}`}>
          클릭
        </button>
      );
    }
    ```
    * `disabled`가 true면 -> `btn btn-disabled`
    * false면 -> `btn false`
    * **주의** : `false && 'xxx'`는 `false`를 반환
      * 보안 : `disabled ? 'btn-disabled' : ''`
    
  * **패턴 3: 배열 + filter + join(여러 조건)**
    ```JSX
    function Button({ variant, size, disabled }
      const classNames = [
        'btn',
        `btn-${variant}`,
        `btn-${size}`,
        disabled && 'btn-disabled'
      ].filter(Boolean).join(' ');
      // false/null/undefined 제거 후 공백으로 합치기

      return <button className={classNames}>클릭</button>;
    }
    ```
</details>

  <summary>(5) CSS Modules - 파일별 스코프 격리(강력 추천⭐️)</summary>

  * **기본 CSS의 문제**
    ```CSS
    /* Button.css */
    .title { color: blue; }

    /* Card.css */
    .title { color: red; }

    // 두 파일의 .title이 충돌함. 하나의 스타일이 앱 전체에 영향을 줌
    ```

  * **CSS Modules의 해결**
    ```CSS
    /* Button.module.css */
    .button { padding: 10px 20px; }
    .primary { background-color: blue; }
    ```
    ```JSX
    import styles from './Button.module.css';

    function Button() {
      return (
        <button className={`${styles.button}${styles.primary}`}>
          클릭
        </button>
      );
    }
    ```
