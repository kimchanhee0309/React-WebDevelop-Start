# 1. JSX

<details>
  <summary>(1) JSX란?</summary>

  * JSX = **JavaScript 속에서 HTML처럼 쓸수 있는 문법**
</details>

<details>
  <summary>(2) JSX의 3가지 기본 규칙</summary>

  * **규칙 1: 반드시 하나의 태그로 감싸기**
    ```JavaScript
    // 에러! 태그 두 개가 나란히 있음
    return (
      <h1>제목<h1>
      <p>내용</p>
    );

    // OK - div로 감쌌음
    return (
      <div>
        <h1>제목</h1>
        <p>내용</p>
      </div>
    );

    // 감싸는 div가 싫다면? 프래그먼트를 통해 해결 가능
    ```
    
  * **규칙 2: HTML 속성은 camelCase로**
    * JSX와 HTML은 비슷하지만, **속성 이름이 약간 다름**
      
  |HTML|JSX|
  |---|---|
  |`class`|`className`|
  |`for`|`htmlFor`|
  |`onclick`|`onClick`|
  |`onchange`|`onChange`|
  |`tabindex`|`tabIndex`|

  ```JavaScript
  // HTML 스타일
  <div class="container" onclick="handleClick()">

  // JSX 스타일
  <div className="container" onClick={handleClick}>

  // class는 JS에서 이미 예약어이기 때문에, 겹치지 않게 바꾸는 것
  ```

  * **규칙 3: 모든 태그는 반드시 닫기**
    ```JavaScript
    // X, HTML에선 가능했지만 JSX에선 에러
    <br>
    <img src="photo.jpg">
    <input type="text">

    // O, 셀프 클로징(/>)으로 닫기
    <br />
    <img src="photo.jpg" />
    <input type="text" />
    ```
</details>

<details>
  <summary>(3) JSX에서 주석 쓰기</summary>

  * JSX 안에선 주석을 `{/* */}` 형태로 씀
    ```JavaScript
    function App() {
      return (
        <div>
        {/* 이렇게 주석을 쓴다 */}
        <h1>제목</h1>

        {/*
          여러 줄 주석도
          이렇게 가능해요
        */}
        <p>내용</p>
      </div>
      };
    }
    ```
</details>

# 2. 프래그먼트(Fragment)

<details>
  <summary>(1) <div>로 감싸면 생기는 문제</summary>

  * **문제 1: 불필요한 DOM이 쌓임**
    * `<div>` 감싸기를 반복하면 실제 HTML 구조는 다음과 같이 됨
      ```JavaScript
      <div>
        <div>
          <div>
            <div>
              <h1>안녕</h1>
            </div>
          </div>
        </div>
      </div>
      ```
      * **의미 없는 `<div>` 껍데기가 잔뜩 쌓임**
      
  * **문제 2: 특정 HTML 구조에선 아예 에러**
    * `<table>` 같이 **규칙이 엄격한 구조**에선 `<div>`를 끼워넣으면 HTML이 깨짐
      ```JavaScript
      // 잘못된 구조
      function TableRow() {
        return (
          <div>    {/* 여기 div 때문에 테이블이 망가짐! */}
            <td>이름</td>
            <td>나이</td>
          </div>
        };
      }
      ```
      ```JavaScript
      <!-- 결과: HTML 표준 위반! -->
      <table>
	      <thead>
	        <tr>
	          <div>   ← table 안에 div가 들어갈 수 없어요!
	            <td>이름</td>
	            <td>나이</td>
	          </div>
	        </tr>
	      </thead>
      </table>
      ```
</details>


<details>
  <summary>(2) Fragment = 투명한 포장지</summary>

  * Fragment는 **투명한 포장지**임
  * 여러 물건을 **묶어주긴 하지만**, 실제 DOM에선 **보이지 않음**
  * `<div>`가 택배 박스라면, Fragment는 **랩으로 살짝 감싸는 느낌**
</details>


<details>
  <summary>(3) Fragment 쓰는 2가지 방법</summary>

  * **방법 1: 빈 태그 `<> </>`(가장 많이 씀)⭐**
    ```JavaScript
    function TableRow() {
      return (
        <>
          <td>이름</td>
          <td>나이</td>
        </>
      };
    }
    ```
  * **방법 2: `<React.Fragment>`(풀네임)**
    ```JavaScript
    import React from 'react';

    function TableRow() {
      return (
        <React.Fragment>
          <td>이름</td>
          <td>나이</td>
        </React.Fragment>
      };
    }
    ```

  * **✨ Fragment를 쓴 결과**
    ```JavaScript
    <!-- 이제 HTML 구조가 깨끗해요! -->
    <table>
      <tr>
        <td>이름</td>
        <td>나이</td>
      </tr>
    </table>
    ```
    * `<div>`도, Fragment도 **결과 HTML엔 흔적이 없음**
</details>


<details>
  <summary>(4) Fragment vs div - 언제 뭘 쓸까?</summary>

  |상황|추천|
  |---|---|
  |스타일(CSS)이나 클래스가 필요할 때|`div`|
  |id, onClick 등 속성이 필요할 때|`div`|
  |단순히 그룹화만 필요할 때|Fragment`<>`|
  |테이블, 리스트 등 HTML 구조가 엄격할 때|Fragment`<>`|

  * **간단한 판단 기준**
    * **이 감싸는 태그에 뭔가 필요한가?**
      * 스타일/속성이 필요해 -> `<div>`
      * 그냥 묶기만 하면 돼 -> `<> </>`

  * **⚠️ 예외: Key가 필요할 땐 `React.Fragment`**
    * **딱 한 가지 경우**에만 빈 태그 `<>` 대신 **풀네임 `<React.Fragment>`**를 써야 함
    * 바로 **리스트에 key가 필요할 때**임
      ```JavaScript
      // 빈 태그는 key를 받을 수 없음
      users.map(user => (
        <key={user.id}>  {/* 이렇게 못 씀! */}
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </>
      ))
      
      // key가 필요하면 풀네임 Fragment 사용
      users.map(user => (
        <React.Fragment key={user.id}>
          <h3>{user.name}</h3>
          <p>{user.email}</p>
        </React.Fragment>
      ))
      ```

  * **정리**
    
  |문법|용도|
  |---|---|
  |`<> </>`⭐|거의 모든 경우(권장)|
  |`<React.Fragment>`|key가 필요한 예외 상황|
  |`<div>`|스타일/속성이 필요할 때|
      
</details>

# 3. JSX에서 JavaScript 사용하기

<details>
  <summary>(1) 변수와 상수 - 기본 중의 기본</summary>

  ```JavaScript
  function UserCard() {
    const userName = '김철수';
    const userAge = 28;

    return (
      <div>
        <h2>{userName}</h2>
        <p>나이: {userAge}세</p>
      </div>
    };
  }

  // 가장 흔한 사용법
  // JSX 바깥에서 변수를 선언하고 안에서 꽂아 넣기
  ```
</details>

<details>
  <summary>(2) 객체 사용 - 점(.)으로 접근</summary>

  ```JavaScript
  function UserProfile() {
    const user = {
      name: '이영희',
      age: 32,
      job: '디자이너'
    };

    return (
      <div>
        <h2>{user.name}</h2>
        <p>{user.age}세, {user.job}</p>
      </div>
    );
  }
  ```
  * 객체의 속성은 **점 표기법(`user.name`)** 으로 접근
</details>

<details>
  <summary>(3) 함수 호출 - 복잡한 로직은 함수로</summary>

  ```JavaScript
  function Greeting() {
    function getGreeting(hour) {
      if (hour < 12) return '좋은 아침입니다';
      if (hour < 18) return '좋은 오후입니다';
      return '좋은 저녁입니다';
    }

    const currentHour = new Date().getHours();

    return <h1>{getGreeting(currentHour)}</h1>;
  }
  ```
  * **조건이 복잡하면 함수로 빼야함**
  * JSXsms **뭘 보여줄지**에 집중하고, **어떻게 계산할지**는 함수에 맡기는 게 깔끔함
</details>

<details>
  <summary>(4) 조건 표현식 - 삼항 연산자</summary>

  ```JavaScript
  function StatusBadge({ status }) {
    return (
      <span className={status === 'active' ? 'badge-green' : 'badge-gray'}>{status === 'active' ? '활성' : '비활성'}</span>
    );
  }
  ```
</details>

<details>
  <summary>(5) 배열 메서드 - map과 filter</summary>

  * 리스트를 화면에 뿌릴 때 가장 많이 쓰는 패턴
  * **기본 - `map` - 배열의 각 요소를 JSX로 변환**
    ```JavaScript
    function FruitList() {
      const fruits = [
        { id: 1, name: '사과' },
        { id: 2, name: '바나나' },
        { id: 3, name: '오렌지' }
      ];

      return (
        <ul>
          {fruits.map(fruit => (
            <li key={fruit.id}>{fruit.name}</li>
          ))}
        </ul>
      );
    }
    ```

  * **응용 - `filter` + `map`체이닝**
    ```JavaScript
    function UserList({ users }) {
      return (
        <ul>
          {users
            .filter(user => user.active)  // 활성 사용자만
            .map(user => (                // JSX로 변환
              <li key={user.id}>{user.name}</li>
            ))}
        </ul>
      );
    }
    ```
  * **⚠️ 안티패턴 #1: `key{index}`**
    ```JavaScript
    // X index를 key로 쓰면 안됨
    {fruits.map((fruit, index) => (
      <li key={index}>{fruit}</li>
    ))}
    ```
</details>
